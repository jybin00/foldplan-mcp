FoldPlan MCP

AI 도구가 내 플래너를 이해하고, 확인한 변경만 안전하게 적용하도록 연결합니다.

FoldPlan MCP는 ChatGPT 같은 MCP 지원 AI 도구에서 FoldPlan의 일정, 목표, 리포트 문맥을 읽고 계획을 도와줄 수 있게 하는 연결 방식입니다. AI가 바로 플래너를 마음대로 고치는 것이 아니라, 먼저 읽고 제안한 뒤 사용자가 확인한 변경만 적용하는 흐름을 목표로 합니다.

사용 흐름 보기 현재 제공 상태
For users: 이 페이지는 FoldPlan MCP를 연결하면 어떤 도움을 받을 수 있는지 설명합니다. Safety first: 쓰기 기능은 항상 미리보기, 사용자 확인, 감사 기록을 거친 뒤에만 적용되는 방향으로 설계됩니다.

AI는 먼저 필요한 플래너 문맥을 읽고, 변경이 필요할 때는 적용 전 미리보기와 확인 단계를 거칩니다.

어떻게 사용하나요?

MCP를 지원하는 AI 도구에 FoldPlan MCP를 연결하면, AI가 사용자의 플래너 문맥을 바탕으로 하루 계획을 설명하고 정리 제안을 만들 수 있습니다.

1. connect

FoldPlan을 AI 도구에 연결

MCP 연결을 통해 AI 도구가 FoldPlan에서 허용된 범위의 플래너 정보를 요청할 수 있게 합니다. 권한은 읽기와 쓰기로 나뉩니다.

2. ask

계획을 물어보고 정리

오늘 일정 요약, 목표 진행 상황, AI 리포트의 근거 같은 정보를 바탕으로 "오늘 뭐부터 하면 좋을까?"처럼 물어볼 수 있습니다.

3. confirm

변경은 확인 후 적용

AI가 할 일을 추가하거나 옮기자고 제안해도 바로 적용하지 않습니다. 바뀔 날짜와 항목을 먼저 보여주고, 사용자가 확인해야 합니다.

AI에게 맡길 수 있는 일

FoldPlan MCP는 플래너를 대신 운영하는 자동화보다, 사용자의 계획을 더 잘 이해하고 정리해주는 보조자 역할에 초점을 둡니다.

read

오늘 계획 이해하기

  • 오늘 플래너 상태와 일정 흐름 요약
  • 목표와 연결된 할 일 파악
  • AI 리포트의 근거가 되는 계획 정보 확인
  • 빈틈, 충돌, 과밀한 시간대 찾기
suggest

수정 제안 만들기

  • 할 일 추가 또는 제목 수정 제안
  • 할 일 완료 상태 변경 제안
  • 일정이나 할 일을 다른 위치로 옮기는 제안
  • 목표 연결처럼 범위가 분명한 메타데이터 정리

플래너 변경은 이렇게 처리됩니다

AI가 만든 변경안은 사용자가 이해할 수 있는 작은 단위의 patch로 제한됩니다. 원본 플래너 데이터를 통째로 바꾸는 방식은 사용하지 않습니다.

  1. 1 변경안 만들기 AI가 "어떤 날짜의 어떤 항목을 어떻게 바꿀지"를 미리보기 형태로 만듭니다.
  2. 2 사용자 확인 사용자는 바뀔 항목과 영향을 받는 날짜를 확인합니다. 확인하지 않은 자동 수정은 기본 흐름이 아닙니다.
  3. 3 안전 검사 후 적용 권한, 확인 토큰, 충돌 여부, 중복 적용 방지, 감사 기록을 다시 확인한 뒤 변경을 적용합니다.
important

AI가 마음대로 지우거나 갈아엎지 못하게 합니다

FoldPlan MCP의 쓰기 흐름은 작은 변경 단위만 허용합니다. 전체 planner JSON 교체, 광범위한 일괄 삭제, 확인 없는 자동 적용, cookie나 URL query token 기반 인증은 허용하지 않는 방향입니다.

개인 계획을 다루기 위한 안전장치

플래너에는 하루 계획, 목표, 실행 기록이 들어갑니다. 그래서 FoldPlan MCP는 필요한 정보만 제한적으로 전달하고, 변경은 다시 확인하는 구조를 둡니다.

scopes

읽기와 쓰기 권한 분리

계획을 읽는 권한과 플래너를 수정하는 권한을 분리합니다. 쓰기 적용 시점에는 권한을 다시 확인합니다.

freshness

오래된 미리보기 방지

제안 이후 사용자가 같은 날짜를 직접 바꿨다면, 기존 미리보기는 오래된 것으로 보고 다시 제안하게 만듭니다.

audit

나중에 확인 가능한 기록

어떤 도구가, 어떤 사용자 권한으로, 어떤 변경을 요청하고 적용했는지 추적할 수 있는 감사 기록을 남기는 방향입니다.

현재 제공 상태

FoldPlan MCP는 단계적으로 열립니다. 사용자는 먼저 읽기 중심의 도움을 받고, 쓰기 기능은 안전한 확인 흐름이 준비된 범위부터 제공됩니다.

available first

플래너 문맥 읽기

AI가 FoldPlan의 제한된 planner metadata, 목표, 리포트 문맥을 이해하도록 돕는 읽기 기반 사용을 먼저 제공합니다.

preview first

변경은 미리보기부터

쓰기 기능은 바로 적용이 아니라 변경안 미리보기, 사용자 확인, 적용 검사 순서로 제공됩니다.

rolling out carefully

지원 범위는 점진적으로 확대

처음에는 작은 할 일 생성처럼 검증 가능한 변경부터 시작하고, 이동, 목표 연결, 시간 블록 같은 기능은 안전장치가 준비된 범위에서 넓혀갑니다.

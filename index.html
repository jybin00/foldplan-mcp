How to Use FoldPlan MCP

AI 도구에서 FoldPlan의 계획, 목표, 리포트를 읽고 안전한 변경 제안을 만들 수 있습니다.

FoldPlan MCP는 MCP를 지원하는 AI 도구가 FoldPlan의 플래너 문맥을 읽고, 사용자가 확인한 변경만 적용하도록 연결하는 인터페이스입니다. 현재는 비공개 베타/개발 접근을 전제로 하며, 공개 self-serve OAuth 연결은 준비 중입니다.

설정 방법 보기 프롬프트 예시
Beta MCP server URL: https://foldplan.com/mcp Current access: 주소는 준비되어 있지만 공개 runtime은 꺼져 있습니다. 초대받은 베타 사용자 또는 개발 환경에서 발급된 mcp_ Bearer token이 필요합니다.

AI는 먼저 필요한 플래너 문맥을 읽고, 변경이 필요할 때는 미리보기와 사용자 확인 단계를 거칩니다.

1. FoldPlan MCP로 할 수 있는 일

AI에게 FoldPlan의 원본 데이터를 통째로 넘기지 않고, 제한된 도구를 통해 필요한 계획 문맥만 읽게 합니다.

read plan

오늘과 이번 주 계획 보기

get_today_plan, get_weekly_plan으로 날짜별 플래너 요약과 상태 메타데이터를 읽을 수 있습니다.

read goals

목표 흐름 확인

get_goal_summary로 active, paused 같은 목표 요약을 읽고 오늘 해야 할 일과 연결해 생각할 수 있습니다.

read reports

AI 리포트 참고

get_latest_ai_report로 FoldPlan 안에서 생성된 최신 리포트 요약을 불러와 계획 판단에 활용할 수 있습니다.

available tools

읽기 도구

  • get_today_plan: 특정 날짜의 planner summary
  • get_weekly_plan: 7일 범위의 planner summary
  • get_visible_ddays: 표시 가능한 D-day 요약
  • get_goal_summary: 목표 목록 요약
  • get_latest_ai_report: 최신 AI 리포트 요약
beta write

쓰기 도구

  • planner.propose_patch: 적용 전 변경안 만들기
  • planner.apply_patch: 확인 토큰으로 변경 적용
  • 현재 검증된 성공 경로는 기존 섹션에 할 일을 생성하는 작은 변경입니다.
  • 삭제, 광범위 수정, 시간 블록 변경은 공개 사용 범위가 아닙니다.

2. 설정 방법

공개 self-serve 연결이 열리기 전까지는 FoldPlan MCP 베타 접근 권한이 필요합니다. 권한이 없으면 mcp_runtime_disabled, mcp_token_exchange_disabled, 또는 인증 오류가 나올 수 있습니다.

before you start

준비물

  • FoldPlan 계정
  • FoldPlan MCP 베타 접근 권한 또는 개발 환경
  • mcp_로 시작하는 FoldPlan MCP access token
  • HTTP MCP server 또는 Bearer header 설정을 지원하는 AI/MCP client
Field Value
Server URL https://foldplan.com/mcp 베타/활성화된 환경에서 사용
Resource URI https://foldplan.com/mcp
Tools list GET https://foldplan.com/mcp/tools
Tool call POST https://foldplan.com/mcp
Auth Authorization: Bearer <FOLDPLAN_MCP_TOKEN>
Read scope mcp.read
Public runtime 현재 공개 self-serve 접근은 꺼져 있습니다. 권한이 없으면 mcp_runtime_disabled가 정상 응답입니다.
  1. 1 FoldPlan에서 MCP access token을 준비합니다 공개 OAuth 연결 전에는 초대받은 베타/개발 환경에서 발급된 토큰이 필요합니다. 토큰은 URL query에 넣지 말고 항상 Bearer header로만 사용하세요.
  2. 2 도구 목록을 확인합니다 아래 요청이 성공하면 해당 토큰으로 FoldPlan MCP 읽기 도구를 사용할 수 있습니다. 
    curl -fsS https://foldplan.com/mcp/tools \
  -H "Authorization: Bearer $FOLDPLAN_MCP_TOKEN"
  3. 3 AI/MCP client에 서버를 추가합니다 client가 custom HTTP MCP와 Bearer header를 지원한다면 다음 형태로 등록합니다. client마다 설정 파일 위치와 필드 이름은 다를 수 있습니다. 
    {
  "mcpServers": {
    "FoldPlan": {
      "type": "http",
      "url": "https://foldplan.com/mcp",
      "headers": {
        "Authorization": "Bearer ${FOLDPLAN_MCP_TOKEN}"
      }
    }
  }
}
  4. 4 직접 호출로 먼저 테스트합니다 client 연결 전에 오늘 계획 읽기부터 확인하면 문제를 좁히기 쉽습니다. 
    curl -fsS https://foldplan.com/mcp \
  -H "Authorization: Bearer $FOLDPLAN_MCP_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "tool": "get_today_plan",
    "input": { "date": "2026-07-03" }
  }'
ChatGPT / Claude

공식 OAuth 연결은 준비 중

ChatGPT 앱, Claude custom connector처럼 OAuth discovery와 client id가 필요한 연결은 FoldPlan의 공개 OAuth/PKCE 연결이 열린 뒤 정확한 버튼과 client id를 안내할 예정입니다.

Cursor / CLI

Bearer header 지원 client부터

Cursor, CLI, 내부 agent처럼 custom HTTP server와 header 설정을 지원하는 도구는 베타 토큰으로 먼저 연결 테스트를 할 수 있습니다.

4. 제한사항과 문제 해결

FoldPlan MCP는 개인 계획 데이터를 다루므로 기능을 한 번에 모두 열지 않습니다. 연결이 되지 않을 때는 아래 항목을 먼저 확인하세요.

not public yet

공개 self-serve 연결은 준비 중

공식 ChatGPT/Claude OAuth 연결은 아직 공개된 상태가 아닙니다. 베타 권한이 없으면 서버 URL을 넣어도 인증 또는 disabled 응답이 나올 수 있습니다.

token safety

토큰은 Bearer header로만 사용

access_token이나 token query string은 허용하지 않습니다. 토큰을 프롬프트, URL, 스크린샷에 붙여 넣지 마세요.

write limits

쓰기 기능은 작은 변경부터

현재 검증된 apply 경로는 기존 섹션에 할 일을 생성하는 흐름입니다. 삭제, 일괄 수정, 시간 블록 변경, raw planner JSON 교체는 지원하지 않습니다.

common errors

자주 보는 응답

  • mcp_runtime_disabled: 해당 환경에서 MCP runtime이 꺼져 있습니다.
  • mcp_token_exchange_disabled: 토큰 발급 API가 공개되어 있지 않습니다.
  • mcp_unauthorized: Bearer token이 없거나 형식이 잘못되었습니다.
  • mcp_insufficient_scope: 토큰에 필요한 scope가 없습니다.
privacy

AI에게 보내지 않는 것

  • 전체 planner JSON 원본
  • 세션 쿠키, raw access token, refresh token
  • Google Calendar token 같은 provider credential
  • 확인하지 않은 planner 변경

FoldPlan MCP

AI assistant가 플래너 문맥을 읽고, 변경은 확인된 제안만 적용하도록 설계합니다.

FoldPlan MCP는 계획 요약, 목표, AI 리포트 같은 플래너 문맥을 외부 AI 도구와 안전하게 연결하기 위한 설명 가능한 인터페이스입니다. 현재 방향은 먼저 읽고, 쓰기는 제안·확인·감사 흐름을 통과한 뒤에만 허용하는 것입니다.

Static explainer: 이 페이지는 GitHub Pages 스타일의 정적 설명면이며, production app route나 foldplan.com 설정을 변경하지 않습니다. Safety stance: write tools are disabled-by-default until scopes, confirmation, audit, and apply integration are ready.

Read tools can return bounded planner metadata. Write tools follow a preview-first sequence before any planner data changes.

Read First

첫 MCP surface는 플래너를 대신 조작하는 것이 아니라, 사용자가 이미 만든 계획 문맥을 안전하게 요약해서 AI assistant가 이해하게 하는 데 집중합니다.

planner.read

오늘 계획 메타데이터

planner day의 bounded summary와 state metadata를 읽습니다. 할 일 원문, private memo/comment, calendar details, 원본 planner JSON은 반환하지 않습니다.

reports.read

근거 있는 리포트

AI 리포트는 planner facts에 묶인 결과를 전달하고, 임의로 새 이야기를 만들지 않도록 출력 경계를 둡니다.

goals.read

목표와 실행 흐름

목표와 실행 기록을 assistant가 참고할 수 있게 하되, 민감한 원문과 불필요한 세부 데이터는 줄입니다.

Write, But Gated

FoldPlan MCP write는 바로 수정하는 단일 버튼이 아니라, typed patch를 미리 보여주고 사용자가 확인한 뒤 적용하는 흐름입니다.

  1. 1 planner.propose_patch Requires scope planner.patch.propose. 변경 의도, 대상 날짜, 예상 결과, 충돌 여부 같은 bounded preview metadata를 만듭니다. rollback/undo data는 preview에 노출하지 않습니다.
  2. 2 User confirmation 사용자가 어떤 날짜와 항목이 바뀌는지 확인합니다. 확인 없는 자동 쓰기는 기본값이 아닙니다.
  3. 3 planner.apply_patch Requires scope planner.patch.apply. confirmation token, idempotency key, stale preview 여부를 다시 검사하고 적용합니다.
disabled by default

왜 이렇게 나눠요?

플래너 데이터는 하루 계획과 실행 기록입니다. MCP client가 실수하거나 너무 넓은 요청을 보내도 사용자가 확인한 typed operation만 적용되도록 해야 합니다. 그래서 apply는 bearer-token auth, scope, affected-date validation, audit identity, idempotency가 함께 맞아야 하고, undo/rollback metadata는 apply 성공 전에 서버 쪽 안전장치로 준비되어야 합니다.

What Can Be Changed

쓰기 도구는 arbitrary JSON replacement가 아니라 FoldPlan이 이해하는 작은 patch family로 제한됩니다.

allowed shape

Typed patch families

  • 할 일 생성, 제목 수정, 완료 토글
  • 요일 또는 날짜 간 할 일 이동
  • 승인된 시간 블록 생성 또는 수정
  • 목표 연결처럼 명확히 범위가 있는 메타데이터 변경
not allowed

Hard boundaries

  • 원본 planner JSON 통째 교체
  • 임의 delete/edit-all 또는 광범위 bulk rewrite
  • cookie 또는 query-string token으로 MCP 인증
  • production write를 기본 활성화하는 설정 drift

Safety Model

FoldPlan MCP는 변경 가능한 능력을 제품 가치로 보지만, 안전 장치를 통과한 변경만 실제 플래너 상태에 닿게 해야 합니다.

scope

명시적 권한

mcp.read, planner.read, goals.read, reports.read 같은 read scopes와 planner.patch.propose, planner.patch.apply write scopes를 분리하고, apply 시점에 다시 확인합니다.

conflict

Stale preview 방지

제안 이후 사용자가 같은 날짜를 바꿨다면 preview를 오래된 것으로 보고 다시 제안하게 만드는 방향입니다.

audit

감사 가능한 적용

user id, MCP client, tool name, operation id, target object ids, confirmation id를 남길 수 있는 구조를 전제로 합니다.
